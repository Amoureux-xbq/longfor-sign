# 🎯 X/Y 坐标计算工具使用说明

## 📝 功能说明

`calculate_xy.py` 是一个独立工具，用于根据背景图和滑块图自动计算验证码的 x（滑动距离）和 y（垂直位置）坐标。

---

## 🚀 使用方法

### 基本用法

```bash
python calculate_xy.py <p1_背景图路径> <p2_滑块图路径> [识别方法]
```

### 参数说明

| 参数 | 说明 | 必需 |
|------|------|------|
| `p1` | 背景图路径（**需要先还原的完整背景图**） | ✅ 必需 |
| `p2` | 滑块图路径（PNG 格式，带透明通道） | ✅ 必需 |
| `method` | 识别方法（见下表） | ❌ 可选，默认 `shape` |

### 识别方法

| 方法 | 说明 | 推荐度 |
|------|------|--------|
| `shape` | 形状匹配（模板匹配） | ⭐⭐⭐⭐⭐ 推荐 |
| `auto` | 自动选择最佳方法 | ⭐⭐⭐⭐ |
| `contour` | 轮廓检测 | ⭐⭐⭐ |
| `color` | 颜色差异 | ⭐⭐ |

## 🐛 常见问题

### Q1: 背景图需要是还原后的吗？

**A:** ✅ **是的！** 必须先使用 `restore_correct.py` 还原背景图，否则无法正确识别缺口位置。

### Q2: 为什么计算出的 x 值比实际位置小 60？

**A:** 这是**正常的**！龙湖滑块图的左侧有 60px 的透明区域，我们计算的是**实际需要滑动的距离**，已经自动减去了这个偏移量。

### Q3: shape 方法和 auto 方法有什么区别？

**A:** 
- `shape`: 强制使用形状匹配，速度快，准确率高（**推荐**）
- `auto`: 综合三种方法的结果，自动选择置信度最高的

### Q4: 如何在脚本中调用？

**A:** 可以通过 Python 导入使用：

```python
from calculate_xy import calculate_xy

x, y = calculate_xy("restored_bg.jpg", "slider.png", "shape")
print(f"x={x}, y={y}")
```
